## M1 — Project Proposal 

Teams submit a proposal containing: 

- Domain and motivation — why does this problem matter? 

- Dataset description — source, size, format, access method, ethical considerations. 

- Scientific question — specific, measurable, and tied to a real decision or outcome. 

- Preliminary hypothesis — what do you expect to find, and why? 

- Roles — who is responsible for what on the team? 

The proposal should be submitted using the GitHub project of the team, using markdown (name of the file: **proposal.md**)



## M2 & M3 — Data Acquisition & Preprocessing (Weeks 3–4)

Graded as a combined submission due at the end of Week 4.

Must include:

- The raw dataset (or a reproducible script to acquire it). 
- A preprocessing log documenting every transformation with justification (file name: **preprocessing.md**). 
- The cleaned, analysis-ready dataset. 
- A data quality summary: null rates, class balance, outlier treatment, and schema (file name: **summary.md**).



## M4 — Exploratory Data Analytics Report

A written report (4–6 pages) plus a Jupyter Notebook with at least 8 visualisations.

Must include:

- Univariate, bivariate, and multivariate analyses. 
-  Written interpretation for every visualisation — a chart without a takeaway does not count. 
-  Revised hypotheses based on what EDA revealed. 
-  A clear statement of the modelling question that the team will pursue.



## M5 — Data Modelling & Visualisation Report

A written report (5–7 pages) plus notebook.

Must include:

- At least two models trained and evaluated with a justified rationale for each choice. 
- Evaluation using at least two metrics appropriate to the problem type. 
- A comparison table or chart showing model performance. 
- At least three stakeholder-facing visualisations communicating the main results. 
- A brief section on model limitations and potential failure modes



## M6 — Final Deployment, Report & Presentation

The complete project package; final delivery. Teams submit:

- Full written report (~10 pages, .md format) covering all project phases. 
- Video of a working demo or deployment (web app, API endpoint, interactive notebook, or recorded walkthrough). 
- A 7-minute class presentation followed by 5 minutes of Q&A. 
- A model card (one page) summarising the model's intended use, performance, and limitations.

The schedule of the presentation will be shared close to the class. **All deliverables should be submitted by the end of the week, on GitHub,** except the 7-minute class presentation slides, which should be on GitHub before the class.
